# Contributing to StreamPipe

Thank you for improving the StreamPipe standard.

## Before opening a change

1. Read `GOVERNANCE.md`, `AGENTS.md`, and the relevant SPSS documents.
2. Search existing issues and ADRs for related work.
3. Keep a proposal limited to one coherent concern.

## Specification changes

Normative changes MUST:

- identify the SPSS document and requirement IDs affected;
- state compatibility impact and migration behavior;
- include examples or test vectors when protocol behavior changes; and
- update `RFC_INDEX.md` when a document status changes.

Use RFC 2119 / RFC 8174 keywords only for objectively testable requirements. A normative requirement MUST use one stable requirement ID.

## Pull requests

Submit focused pull requests. Do not mix editorial cleanups with a semantic protocol change. The pull-request template is required; reviewers may request an ADR for a material architectural decision.

## Code of conduct

Be constructive, respectful, and specific. Technical disagreement is welcome; personal attacks and harassment are not.
